The Importance of Zero-Knowledge Privacy

This tool allows you to create notes that are encrypted in your browser before ever reaching our servers. In an era of mass surveillance and data breaches, sending sensitive information (like passwords or credit card details) over standard chat apps is risky.

By using AES-256 encryption, this tool ensures that only someone with the correct password can read the note. Even if our database were hacked, the attackers would only see scrambled gibberish.

🛠️

How to Use the Encrypted Note Tool

  1. Log In: To generate shareable 'burn-after-reading' links or to save your encrypted notes to the cloud, please log in.
  2. Type your sensitive message in the text area.
  3. Enter a strong password. Do not forget it; we cannot recover it.
  4. Click Encrypt and copy the resulting link or raw text to send to your recipient.
✨

Key Features & Benefits

  • Client-Side Encryption: Your password never leaves your computer. We cannot read your notes.
  • Burn-After-Reading: Logged-in users can create links that self-destruct after being opened once.
  • Password Protection: Add an extra layer of security to prevent unauthorized access.
  • Expiry Timers: Set notes to automatically delete after 1 hour, 1 day, or 1 week.
💡

When to Use Dead Drop Notes

This tool is ideal for sending sensitive data securely:

🔑 Password Sharing

Send a temporary password to a client or team member securely without leaving a permanent record in your chat history.

💳 Credit Card Details

If you must share payment information with a trusted person, use a self-destructing note to minimize risk.

🕵️ Whistleblowing

Journalists and sources use zero-knowledge tools to exchange information without revealing their identity.

🚀

Why a Login is Required for Cloud Storage

Generating a random encryption key and scrambling text in your browser is free and requires no account. However, if you want to store that encrypted note on our servers so you can send a link to a friend, we must host that data. We require a login for these features to:

  • Manage the Database: We need to link the encrypted payload to your account so you can delete it or change the expiration time.
  • Enforce the "Burn" Logic: Our server must track whether the link has been clicked yet to know when to delete the data forever.
❓

Frequently Asked Questions (FAQ)

Q: Can you recover my password? ▼
No. Because the encryption happens locally, we never see your password. If you lose it, the note is gone forever.
Q: Is login required to use the tool? ▼
Yes, a free account is required to generate shareable 'burn-after-reading' links or to save your encrypted notes to the cloud. Local encryption is free.
Q: What happens when a note "burns"? ▼
The database record is completely erased. Anyone visiting the link after that will see a 404 error.
